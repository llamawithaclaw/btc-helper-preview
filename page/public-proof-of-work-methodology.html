BTC Helper Public Proof-of-Work Methodology

This page explains what the public proof-of-work track is trying to show, what it does and does not claim, and how the benchmark should be interpreted.

Purpose

This document describes the methodology behind BTC Helper's public proof-of-work tracking system.

Core Principle

BTC Helper's public proof-of-work is not about predicting price. It is about demonstrating operational discipline and run usability over time.

What We Track

1. Daily Regime Posture

  • Regime: The current accumulation posture (Aggressive, Normal, Cautious, Pause)
  • Action: The recommended accumulation behavior (Increase, Maintain, Be Selective, Pause)
  • Confidence: Run usability rating (High, Moderate, Reduced, Unknown)

2. Portfolio Performance Benchmark

  • BTC Helper Portfolio: Value of a hypothetical portfolio following BTC Helper's accumulation policy
  • Baseline DCA Portfolio: Value of a hypothetical portfolio following simple dollar-cost averaging
  • Benchmark Difference: Percentage difference between the two strategies

3. Data Quality Metrics

  • Coverage: Percentage of required fields with live data
  • Import Reliance: Number of fields coming from imported/manual-assisted sources
  • Run Quality: Overall usability rating (Live, Degraded, Recently Recovered, etc.)

Methodology Details

Portfolio Tracking

The benchmark uses a simple model:

  • Starting capital: $1,000 (or equivalent in BTC)
  • Baseline DCA: Buys $10 every day regardless of market conditions
  • BTC Helper: Adjusts daily buy amount based on regime:
  • Aggressive: $15 (50% increase)
  • Normal: $10 (baseline)
  • Cautious: $5 (50% decrease)
  • Pause: $0 (no buy)

Data Sources

  • Price: CryptoCompare API (daily close)
  • On-chain metrics: Glassnode-style historical references
  • Derivatives: OKX or CoinGlass-style references
  • Macro: FRED-style references

Validation Approach

  1. Execute the daily run with current live or mixed-source data
  2. Evaluate regime behavior against scenario expectations
  3. Log domain breakdown and data-quality caveats
  4. Publish output with confidence and coverage framing

Transparency Commitments

1. Honest Degradation Reporting

When data is incomplete or imported, this is clearly stated in the daily output.

2. Clear Methodology Notes

Benchmark calculations should be understandable and reproducible at the level described here.

3. Public Archive

Historical posture posts are preserved in the public archive.

4. No Performance Theater

BTC Helper does not claim predictive certainty or guaranteed outperformance. The public proof-of-work is meant to demonstrate consistency, explainability, and operational honesty.

Interpretation Guide

When Benchmark Difference Is Zero

This means BTC Helper and baseline DCA are currently performing identically. That can happen when the regime remains near baseline or the sample is still short.

When Benchmark Difference Is Positive

This means BTC Helper is outperforming baseline DCA so far.

When Benchmark Difference Is Negative

This means BTC Helper is underperforming baseline DCA so far.

Sample Size Warning

Early readings, especially in the first 30 days, should be interpreted cautiously. The benchmark needs a much longer sample to become meaningfully informative.

Future Enhancements

  • Extended backtesting over historical regimes
  • Multiple benchmark variants
  • Drawdown and risk metrics
  • Better visualization of regime transitions

Not the Goal

  • Short-term price prediction
  • Guaranteed outperformance claims
  • Black-box optimization theater
